# MUI Dense, Plan11 Outcome

This pass fixed the large-`cellBlockPadding` clipping bug and narrowed the reusable `DenseDataGrid` API at the same time. The dense library no longer treats `headerFilters` or `headerFilterHeight` as dense policy. `DenseDataGridOptions`, `DenseSettings`, and the reusable presets now only carry grid density plus the CSS-length `cellBlockPadding` knob. The gallery still exposes a local "Show header filters" control, but it now passes that feature prop directly to `DenseDataGrid` instead of threading it through `DenseSettings`.

The clipping bug turned out to have two causes. The obvious one was that the wrapper was applying `paddingBlock` directly to the fixed-height outer `.MuiDataGrid-cell` and `.MuiDataGrid-columnHeader` slots, which let large values fight MUI X's own `line-height` and `overflow: hidden` rules. The less obvious one was that MUI X multiplies explicit row and header heights by its density factor. The wrapper's metrics were therefore describing one box while the rendered grid was drawing a smaller one whenever density was `compact` or `comfortable`.

The final implementation fixes both sides of that mismatch. `cellBlockPadding` now stays a measurement input rather than a literal outer-slot padding rule, and `DenseDataGrid` compensates for MUI X's density scaling before passing `rowHeight` and `columnHeaderHeight` back into `DataGridPro`. That keeps the rendered row height, column header height, and default header-filter height aligned with the measured numbers the wrapper publishes. Because MUI X already defaults `headerFilterHeight` to `columnHeaderHeight`, removing the dense-level filter-height knob did not require inventing a replacement abstraction.

The regression coverage is stronger than before. The direct browser test now checks that the rendered row and header heights match the published metrics and that a large `20px` padding case keeps cell text, header text, row checkboxes, header checkboxes, and header-filter inputs within their visible slots. The gallery page test was updated to match the narrower sidebar surface by removing the old header-filter-height control expectations.

One boundary remains deliberate: if an app passes `headerFilterHeight` directly to `DenseDataGrid`, that value still follows normal `DataGridPro` behavior rather than a dense-specific normalization layer. This pass only compensates the wrapper-owned auto heights.
