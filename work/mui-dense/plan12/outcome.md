# MUI Dense, Plan12 Outcome

This pass made `src/mui-dense/lib/` look more like a reusable local library and less like a gallery host. `DEFAULT_DENSE_SETTINGS` now lives with `DenseSettings` in `src/mui-dense/lib/settings.ts`, and the old `types.ts` / `presets.ts` split is gone. The gallery preset table moved fully into `src/mui-dense/gallery-dense.ts`, along with the preset type. The reusable `lib` surface now exports the baseline settings object, the theme builder, and the grid wrapper, but not the gallery preset table.

The grid settings naming is smaller and clearer than before. The exported `DenseDataGridOptions` and `DenseDataGridDensity` names are gone. `DenseSettings['dataGrid']` now carries the density union inline, and the old `cellBlockPadding` field was first renamed to `contentVerticalPadding` and then tightened again to `textVerticalPadding` once the wrapper's automatic height policy became explicitly text-driven. That name now flows through the library, gallery adapter, sidebar controls, tests, and README. I kept a local `DenseDataGridSettings` alias inside `data-grid.tsx` because the file genuinely reads better with one internal name for that sub-shape, but it is no longer part of the public surface.

The zero-padding fix required changing the measurement model, not just renaming the knob. The old height math added both an `xHeight` term and a checkbox-padding fudge factor, so `0px` still baked in extra slack. The current implementation now makes the boundary explicit: automatic row and header height are based on the measured text line plus the caller-requested text padding, and non-text content like selection checkboxes no longer participates in that height floor. That drops the fragile checkbox-probe heuristic entirely and makes the auto-sizing rule easier to understand. The old `xHeight` and `textLineHeight` diagnostics are gone as well, since they no longer informed any behavior, and the one-field fallback tuning type was collapsed to a plain constant.

The direct browser coverage now checks the behavior that motivated the change. There is a zero-case regression test proving that `0px` produces shorter row and header heights than `1px`, plus the existing large-padding regression test now uses the renamed setting and still verifies that text, checkboxes, and header-filter inputs stay inside their visible slots. The app-level test was updated to the renamed sidebar field and to stop hard-coding the old 30px row/header numbers.

One deliberate boundary remains: presets are now clearly gallery-owned, but the gallery still uses them to exercise the same top-level `createDenseTheme` and `DenseDataGrid` APIs that a real app would call. That keeps the reusable library smaller without losing the proof point the gallery exists to provide.
